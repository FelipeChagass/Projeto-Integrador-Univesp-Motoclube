**Decisions**
- Escopo inclui apenas as Sprints 1-4 pedidas no último pedido do usuário. A Sprint 5 sobre Supabase Auth fica fora deste plano.
- O logout completo do sistema fora de Trocar Membro permanece com o comportamento atual, inclusive limpeza ampla de estado local. A preservação do caixa deve ocorrer apenas no fluxo de Trocar Membro.
- A correção da Sprint 4 deve evitar refatoração grande no backend. O backend atual de caixa já oferece um endpoint suficiente para reconciliação.
- O texto Sair do modo estoque deve ser aplicado nos pontos visíveis de entrada do modo estoque para manter consistência entre desktop e mobile.

**Further Considerations**
1. Se o botão Abrir Caixa também existir no menu lateral apenas como atalho secundário, decidir se ele será mantido ou removido. A leitura atual mostra redundância explícita no header; a recomendação mínima é remover só o header para respeitar estritamente o pedido.
